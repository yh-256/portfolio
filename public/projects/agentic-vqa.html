Video QA

長時間映像に対し、質問を分解し、必要なツールと根拠を選び、回答を検証・再生成する映像質問応答システム。A long-form video QA system that decomposes questions, selects tools and evidence, then verifies and revises answers.

Status
研究・評価中Research in progress
Role
設計 / 実装 / 評価Design / implementation / evaluation
Stack
Python, vLLM, Qwen3-VL, RAG
Focus
Planning, evidence, verification

01 / Context

映像を見られるだけでは、長時間映像には答えられない。Seeing video is not enough for answering long-form video.

長時間映像に対する質問では、全フレームを一度にモデルへ渡すことは難しく、質問ごとに必要な証拠も異なります。物体、動作、音声、時間関係など、問いに応じて確認対象を切り替える必要があります。For long videos, sending every frame to a model is impractical, and each question requires different evidence—objects, actions, speech, or temporal relationships.

そこで、単一のVLMに即答させるのではなく、回答計画を立て、ツールを使い、集めた根拠を検証してから回答するエージェント構成を採用しました。Instead of asking one VLM to answer immediately, the system plans, calls tools, verifies gathered evidence, and only then produces an answer.

02 / Design

役割を分け、失敗箇所を観測可能にする。Separate responsibilities so failures can be observed.

  1. Evidence planning
    質問から必要なモダリティ、時間範囲、検索方法を決定。Decide modalities, time ranges, and retrieval methods from the question.
  2. Tool execution
    フレーム検索、字幕・音声、物体・行動情報などを必要な分だけ取得。Retrieve only the needed frames, speech, objects, or action cues.
  3. Evidence formatting
    根拠を出典・時刻・信頼度とともに共通形式へ整理。Normalize evidence with source, timestamp, and confidence.
  4. Generation & verification
    候補回答を生成し、根拠との整合性を確認。弱い場合は再計画。Generate candidate answers, verify consistency, and replan when weak.

設計上の要点は、エージェントらしさを「自由な会話」ではなく、観測可能な意思決定と再試行として定義したことです。The key design choice is defining agency not as free-form conversation, but as observable decisions and controlled retries.

03 / Evaluation

最終精度だけでなく、設計が効いた理由を検証する。Evaluate not only accuracy, but why the design works.

ベースラインとの総合比較に加え、ツールなし、全ツール固定、計画なし、検証なし、再生成なしといったアブレーションを行います。これにより、単に強いモデルを使った結果ではなく、各エージェント機構の寄与を分離します。Alongside baseline comparisons, the evaluation removes tools, planning, verification, and regeneration one at a time to isolate the contribution of each mechanism.

  • 回答精度と根拠整合性Answer accuracy and evidence consistency
  • ツール呼び出し回数・レイテンシTool calls and latency
  • 質問タイプ別の失敗分析Failure analysis by question type
  • 再計画・再生成が有効だった事例Cases where replanning or revision helped
